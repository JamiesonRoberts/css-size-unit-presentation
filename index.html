Atomic Web Design

What Atomic is not

Its not a framework, naming convention, boilerplate, etc.

IE - BEM, OOCSS, Bootstrap

What Atomic is

Atomic design is methodology for organizing and creating systems

or, in our case a way of structuring our components and what influces what.

Forces you to think about levels of inheritance, and how larger elements interact with smaller elements

Each level should be standalone

Atoms

Like matter, basic building blocks of a website.

Doesn't mean you need to have a partial for each html element tag

Infact you probably shouldn't.

Grid, typography, forms, animations, normalize, etc

Example

input, label, title

Molecules

Smallest fundamental units of a site

Deals with the interaction of atoms and creates a small standalone object

Should be quickly testable and deal with some responsive

Example

A search form with a title.

Organisms

Group of molecules

Its a group that are joined together and form a relativly complex distinct section of an interface

Can contain disparate molecules, or the same one repeated

Header, footer, section

Should be fully responsive

Example

A header with a logo, navigation, and search bar

Templates

Group of organisms AKA a wireframe

Workable interface

Deals with the interaction of organisms

Should incorporate all the ideas of the previous levels

Example

A wireframe with/without placeholders





Pages

Specific Instance of a template

Its the specific instance of page containing content

Should be a functional web page

Example

A page.

Why?

Way of structuring

Language independant while providing a way of structuring without spcific naming convetions

Normalizes file structure so that each type of technology being used uses the same structure

Segmented code

Makes easier to segment code to prevent un-intended interactions either by code or by others programming on the same file

Defines levels of interactions

Each layer deals with specific interactions, and means it should be easier to trace down what is affecting what
